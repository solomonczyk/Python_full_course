# Commercial Beta FAQ

## What is Python Quest?
Python Quest is a story-driven, mission-based Python course for beginners aged 10–16. Instead of traditional video lessons, students learn by completing coding missions within a steampunk narrative, guided by characters Ва (mentor), Новичок (learner), and Багус (bug creature). Every lesson ends with writing real Python code that is checked instantly.

---

## Is this a finished product?
**No.** Python Quest is a **commercial beta**. The product is feature-complete and has been tested with 58 real students (0 critical blockers). However, it is not yet a mass-market polished product. Some lesson wording, mission pacing, and UI details may be adjusted based on your feedback. By joining the beta, you become an active contributor to the product's development.

---

## Who is it for?
Children and absolute beginners aged **10–16** who want to learn programming from zero. It is also suitable for older teenagers and adults who prefer a more engaging, story-driven introduction to Python over dry textbooks. No prior programming experience is required.

---

## Does the student need to know Python already?
**No.** Python Quest assumes zero programming experience. The first lessons explain what a variable is, what data types are, and how to write the first line of code. Absolutely no prior knowledge is required.

---

## Does the student need to install Python?
**No.** Everything runs in the browser. The student writes code in an embedded editor, and the Mission Checker evaluates it instantly. No installations, no PATH configuration, no environment setup. A modern browser (Chrome, Firefox, Edge, Safari) is all that is needed.

---

## How does progress work?
Progress is saved to the backend after each completed lesson and mission. If the student switches devices or clears browser data, re-entering the beta code on the landing page restores all progress. Progress is visible as a quest grid with completion markers, skill tree unlocks, and a clear "what's next" indicator.

---

## What is a beta code?
A beta code is a unique 64-character hex string that serves as your access credential. It is **not** a password — it is a one-time access token. Enter it on the landing page to unlock beta access and restore your progress. If you lose your beta code, contact the team for a reissue.

**Important:** Keep your beta code private. Anyone with the code can access the course and see the progress associated with it.

---

## Is personal child data collected?
**No.** The beta system does **not** ask for or store:
- Child's name, email, or phone number
- Precise age or date of birth
- Home address or school name
- Any government identifier
- Photos, recordings, or biometric data

The only identifier stored is a **hashed version** of the beta code, which cannot be reversed to the original code. Analytics events are anonymous. Feedback forms are submitted by parents/operators, not directly by children.

---

## Is payment already automated?
**No.** Payment is handled manually:
1. You express interest or receive an invitation.
2. You receive a payment link (Stripe or similar).
3. After payment confirmation, your beta code is delivered.
4. The product itself has **no payment UI, no pricing page, and no subscription management**.

This approach ensures that no payment data ever touches the learning environment.

---

## What happens after beta?
After the 12-week beta access period:
- You will be informed whether the full course has launched or the beta has been extended.
- If the full course has launched, existing beta participants will receive instructions on how to transition (including any applicable discounts or progress carryover).
- If the beta is extended, your access may be renewed at no additional cost.

**There is no automatic renewal, no surprise charge, and no lock-in.**

---

## What if the student gets stuck?
- The Mission Checker provides instant, friendly feedback when code is incorrect.
- Characters (Ва, Новичок, Багус) offer hints and encouragement.
- Students can retry missions unlimited times — no penalties, no score deductions.
- If the student is consistently stuck, parents can use the feedback channel to report the issue. The team reviews and responds within 2 business days.
