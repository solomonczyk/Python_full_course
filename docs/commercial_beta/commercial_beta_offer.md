# Python Quest Commercial Beta Offer

## Short Offer

**Get early access to Python Quest** — a story-driven, mission-based Python course for beginners — at a controlled beta price, with full progress persistence, feedback channel access, and the ability to continue from where you left off when the full course launches.

---

## Product Positioning

### What Python Quest Is
Python Quest is an interactive, browser-based Python course for beginners aged 10–16. Instead of passive video lessons, students learn by completing coding missions within a steampunk narrative, guided by three characters: **Ва** (mentor), **Новичок** (fellow learner), and **Багус** (the bug creature). Every lesson ends with writing real Python code that is checked instantly by the Mission Checker.

### Why It Is Different
- **Story-driven learning:** A coherent steampunk narrative runs through all lessons — the student is not "studying Python" but "exploring a world powered by code."
- **Character-guided explanation:** Three perspectives make concepts click: Ва explains, Новичок asks the "dumb" questions, Багус breaks things.
- **Missions, not passive theory:** Every lesson ends with real coding. No "watch and forget."
- **Mistake-friendly:** Errors are met with hints, not penalties. Retries are expected.
- **Real Python, not blocks:** Students write real Python syntax from lesson one.

### Who It Is For
- **Children and beginners aged 10–16** who want to learn programming but find traditional courses boring or intimidating.
- **Parents** looking for a constructive, screen-positive activity that teaches real skills.
- **Tutors and educators** evaluating the product for potential classroom or club use.

### Who It Is Not For
- **Advanced Python developers** — the course covers fundamentals only (variables through dictionaries and simple algorithms).
- **Learners who want only dry reference material** — Python Quest is narrative-driven; if you want a textbook, this is not it.
- **Learners who do not want interactive practice** — every concept requires writing code in missions.

## Buyer Promise

### What the Student Should Achieve
After completing the beta period (Parts 1–3 currently available), the learner should be able to:
- Understand variables, data types, conditions, loops, functions, lists, and dictionaries.
- Write simple Python scripts that take input, make decisions, repeat actions, and use basic data structures.
- Read and fix basic Python errors with confidence.
- Explain what their code does to a peer or parent.
- Feel motivated to continue to the full course.

### What the Parent Should See
- The student is engaged in a constructive, screen-positive activity.
- The student is writing real Python code — not dragging blocks or watching videos.
- Progress is visible: completed lessons, unlocked skills, and clear next steps.
- The product respects privacy: no child personal data is collected.

### What Is Not Promised Yet
- **Full course completion** — the beta covers fundamentals; advanced topics are not available yet.
- **Parent dashboard** — not available in beta. Progress visibility is limited to what the student sees.
- **Certificate of completion** — not available in beta.
- **Lifetime access** — beta access is time-limited (12 weeks). Terms for post-beta access will be communicated separately.
- **Instant support** — feedback is reviewed within 2 business days, not in real time.

## Beta Limitations

### Not a Full Mass Launch
This is a controlled commercial beta, not a mass-market launch. Access is limited to a controlled group of beta participants. The product is validated but not yet polished for scale.

### No Parent Dashboard Yet
Parents cannot create their own login, view student progress from their own device, or receive automated progress reports. Progress is visible only inside the student's session.

### No Full Account System Yet
There is no email/password registration, no OAuth login, and no persistent user profile. Access is managed via beta codes, which are long hex strings stored locally.

### No Automated Payment Layer Yet
Payment is processed manually (invitation/purchase order). There is no self-serve payment flow, no Stripe integration, no subscription management, and no automated refund system. Payment is **not** handled inside the product.

### Product May Change After Feedback
Beta participants acknowledge that the product is in active development. Lessons, missions, dialogue, and progression may be adjusted based on feedback. Some features may be added, removed, or changed during the beta period.
