## Issue Category Breakdown

### Category 1: Dialogue Premature Concept — Natural Russian Words (23 issues)

**Status: All 23 are FALSE POSITIVES**

**Issues:** PQA-0001 through PQA-0023

The audit flags dialogues containing the Russian words `если` (if), `пока` (while/until), `повтор` (repeats), and `услови` (condition) as references to Python keywords (if, while, for) that haven't been taught yet.

**Operator analysis:**

- `если` (PQA-0002–PQA-0017, PQA-0019) is a fundamental Russian conjunction meaning "if" in everyday speech. It appears in phrases like "если я напишу" (if I write), "если хочешь" (if you want), "если введут букву" (if someone enters a letter). These are quotidian language uses, not references to Python's `if` statement, which isn't even syntactically similar. Stripping `если` from dialogue would make all characters speak unnaturally.

- `пока` (PQA-0006, PQA-0007, PQA-0010, PQA-0020–PQA-0023) in these usages means "until/while" in the temporal sense: "ждать, пока..." (wait until...), "пока не разобрался" (until I figured it out). This is an everyday temporal conjunction, not a reference to Python's `while` loop. The fixed expression "пока не разобрался" appears in multiple Bagus lines as a character idiom.

- `повтор` (PQA-0001) means "repeats" — describing what `print()` does on screen. This has nothing to do with Python's `for` loop keyword.

- `услови` (PQA-0018) — acceptable (not false positive; see below).

### Category 2: Dialogue Premature Concept — Acceptable Pedagogical Scaffolding (1 issue)

**Status: ACCEPTABLE**

- **PQA-0018** (1-7, "услови"): Va says "После if всегда ставь двоеточие. Это как знак: дальше команды для этого условия." Lesson 1-7 teaches comparisons (==, >, <, True/False). The `if` keyword preview in the post-error dialogue bridges naturally to lesson 1-8 (where `if` is formally introduced). This is standard scaffolding — not confusing, and pedagogically valuable.

### Category 3: Dialogue Premature Concept — Forward Reference to dict (1 issue)

**Status: ACCEPTABLE**

- **PQA-0024** (4-17, "словар"): The student character asks about sorting "по возрасту из словаря" (by age from a dictionary). Dict is taught in 5-1, making this a forward reference. However, `словарь` in everyday Russian can mean an address book/phonebook, and the character is ASKING (not being taught). This is a natural, curious question that motivates the `key=` parameter. It would not confuse students, who would understand the general concept of a "dictionary" as a lookup table.

### Category 4: Suspicious One-Line Code (2 issues)

**Status: Both FALSE POSITIVES**

- **PQA-0025** (1-1, `print("Привет")`): This is a valid one-line solution for lesson 1-1 where ALL solutions are single-line print statements. The audit's multi-line formatting rule doesn't apply to inherently single-line solutions.
- **PQA-0026** (4-10, `print("\n".join(["a","b"]))`): This is idiomatically correct Python — using `join()` in one line to produce multi-line output. There's no formatting defect.

### Category 5: Wording Ambiguous — "выведи результат" (13 issues)

**Status: 11 ACCEPTABLE, 2 FALSE POSITIVES**

All 13 issues flag `"выведи результат"` as ambiguous. Operator analysis:

- **PQA-0030 (2-3)** — **FALSE POSITIVE**: The task explicitly states `"выведи результат в формате: 'Бросок: X + Y = Сумма'"`. The format IS specified. The audit missed this.
- **PQA-0040 (4-23)** — **FALSE POSITIVE**: The task literally says `"Выведи результат a is b"` — it tells the student exactly what expression to output. No ambiguity.
- **PQA-0027, PQA-0028, PQA-0032, PQA-0033** — **ACCEPTABLE**: Each gives the expected result in parentheses next to the task description (e.g., "(9)", "(7.0)", "(10)"). The student knows what number to expect.
- **PQA-0034 through PQA-0039, PQA-0041** — **ACCEPTABLE**: Each task is contextually clear. The lesson topic (remove, sum, count, extend, sort, function return) makes the expected operation and result unambiguous. All tasks are simple enough that "выведи результат" following the described operation is obvious.

### Category 6: Wording Overstated — "напиши программу" (2 issues)

**Status: Both ACCEPTABLE**
