# Python Quest — Lesson Style Guide

> Версия: 1.0  
> Дата: 2026-06-01  
> Назначение: единые правила для всех 92 уроков курса Python Quest

---

## 1. Структура урока

Каждый урок следует пятишаговой структуре:

```
1. Сцена / Аналогия — вход в тему через образ
2. Диалог — персонажи обсуждают тему
3. Код — рабочий пример
4. Объяснение — что и почему
5. Практика — quiz, what_outputs, find_bug, mission
```

### Порядок подачи

**Правильно:** аналогия → диалог → код → объяснение → практика  
**Неправильно:** сухое определение → пример → тест

> Пример: Для темы `if` сначала идёт образ швейцара в клубе, затем диалог, где Новичок переспрашивает, затем код с `if`, затем разбор "почему это так работает", затем практика.

## 2. Аналогии по темам

### 2.1 Переменные — «сундук / коробка / контейнер с именем»
- Переменная = сундук с биркой
- `name = "Ксю"` = положили ценность в сундук
- `print(name)` = заглянули в сундук
- Без кавычек — имя сундука, в кавычках — содержимое

### 2.2 `print()` — «голос / эхо / глашатай»
- `print()` = крик в пещере, эхо возвращает
- `print("Привет")` = крикнул «Привет» — услышал «Привет»

### 2.3 `input()` — «звонок в дверь / вопрос / запрос»
- `input()` = звонок в дверь, программа ждёт ответ
- `name = input()` = позвонил, тебе открыли, ты назвал имя
- `input()` всегда возвращает строку — даже если ввёл число

### 2.4 `if` / `else` / `elif` — «швейцар / страж / мост / развилка / ворота»
- `if` = строгий швейцар: проверяет пропуск и пускает или нет
- `if/else` = развилка на дороге: один путь или другой
- `elif` = мосты со стражами: первый пропустил — остальные не спрашивают
- Условие = вопрос, на который можно ответить только «да» (`True`) или «нет» (`False`)

### 2.5 Циклы — «конвейер / поезд / маршрут / патруль»
- `for` = конвейер на фабрике: каждая коробка проходит через одну операцию
- `for i in range(n)` = поезд на рельсах: станция 0, станция 1, станция 2...
- `range(start, stop)` = автобусный маршрут: едет от остановки start до stop-1
- `while` = микроволновка: гудит, пока таймер не обнулится
- Бесконечный цикл = таймер без кнопки «стоп»

### 2.6 Функции — «пульт / кнопка / инструмент / рецепт»
- `def` = новая кнопка на пульте: нажал — сработала
- Функция = рецепт: записали однажды, используем многократно
- `return` = выход из функции с результатом (как тостер выпрыгивает с тостом)

### 2.7 Списки — «полка с отсеками / карманы рюкзака»
- Список = полка с пронумерованными ячейками
- Индекс = номер ячейки (всегда с 0)
- `append()` = положил вещь в конец полки
- `len()` = сколько вещей на полке

### 2.8 Словари — «телефонная книга / карточка / паспорт / анкета»
- Словарь = телефонная книга: ищешь по имени, находишь номер
- Ключ = имя в книге, значение = номер телефона
- `KeyError` = ищешь имя, которого в книге нет

### 2.9 `try`/`except` — «страховка / защитная сетка / парашют»
- `try/except` = страховочный трос альпиниста
- `try` = идём по скале, `except` = трос ловит при падении

## 3. Персонажи: стабильные роли

### Ксю (ksyu)
- **Роль:** спокойный наставник, объясняет просто
- **Говорит:** понятными бытовыми аналогиями, без жаргона
- **Не говорит:** сухих определений, сложных терминов без пояснения
- **Пример:** «Представь, что ты стоишь в огромной пещере...»

### Ва (va)
- **Роль:** логический / структурный помощник
- **Говорит:** про правила, порядок, типы, синтаксис
- **Не говорит:** мотивационных фраз, аналогий из быта
- **Пример:** «Умножение и деление выполняются до сложения и вычитания.»

### Да (da)
- **Роль:** проводник в миссиях и практике
- **Говорит:** преимущественно в миссиях, про цель, вызов
- **Не говорит:** в post_error_dialogue (там он редкий гость)

### Ксю — Да (различие)
- Ксю объясняет *как*, Да объясняет *зачем* и *что делать дальше*

### Багус (bagus)
- **Роль:** ошибки, баги, предупреждения, ловушки внимания
- **Говорит:** про ошибки и их последствия
- **Доля в post_error_dialogue:** ~30-50% реплик
- **Не говорит:**  
  - одинаковых фраз в разных уроках
  - общих мотивационных фраз («Ошибка — это тоже прогресс!»)
  - «Ой-ой! Багус нашёл багус!» — только 1 раз во всём курсе
- **Правильные реплики Багуса:**  
  - конкретная реакция на конкретную ошибку  
  - предупреждение о типичной ловушке  
  - короткое замечание с юмором, но по делу

### Новичок (novice)
- **Роль:** задаёт confuse-ные, но полезные вопросы
- **Должен звучать как:** ученик, который *думает*, а не как учитель, подводящий итог
- **Запрещённые паттерны:**  
  - ❌ «А, понял! `input()` возвращает строку. Если нужно число — заворачивай в `int()`.»  
  - ❌ «Осознал: порядок `elif` важен...»  
  - ❌ «Понял: `//` отбрасывает остаток...»  
- **Правильные реплики Новичка:**  
  - ✅ «То есть... (переспрашивает своими словами)»  
  - ✅ «А если я... (пробует вариант)»  
  - ✅ «Значит, получается, что... (сомневающееся понимание)»  
  - ✅ «Погоди, я не понял: а почему... (конкретный вопрос)»  
  - ✅ «А, то есть `кавычки` — это этикетка, а без них — имя переменной?»
- **Новичок НЕ подводит итог урока —** это делают Ксю или Ва

## 4. Качество диалогов

### 4.1 Запрещённые повторы
- Одна и та же реплика не должна встречаться в разных уроках
- Bagus не должен повторять одну фразу дважды в одном диалоге
- Шаблон «А, понял!» — убрать из всех уроков

### 4.2 Живой язык
- Короткие предложения (до 15 слов)
- В вопросах — естественные паузы, сомнения
- Диалог — это не лекция, а беседа
- Персонажи могут перебивать, переспрашивать, уточнять

### 4.3 Структура post_error_dialogue
- Сначала Новичок — его ошибка и осознание (естественное, без «Понял:»)
- Затем Ксю или Ва — объяснение правильного варианта
- Затем Багус — предупреждение или ловушка (если уместно)
- Завершение — коротко, без повторения всей теории

## 5. «Почему это важно» (why this matters)

**Формат:** «Это нужно, потому что...»  
**Где размещать:**  
- `connection_to_game` — прямая связь с финальной игрой  
- `game_relevance` — мотивация: «Ты используешь это, когда...»  
- `mini_summary` — краткий итог: «Это твой инструмент для...»

Каждый урок должен содержать явный ответ на вопрос «Зачем мне это учить?».

### Примеры
- **print:** «Это голос твоей программы. Без него игра будет молчать.»
- **if:** «Это выбор в игре: открыта дверь или нет, жив герой или повержен.»
- **for:** «Это автоматизация: не пиши одно и то же 100 раз.»
- **def:** «Это твой инструмент: собери сложное действие в одну кнопку.»

## 6. Миссия

Каждая миссия должна:
1. Прямо проверять Python-конструкцию урока
2. Иметь достижимый expected_output для новичка
3. Быть связана с игровым сюжетом (арена, башня, инвентарь)
4. Выполняться за 2-8 строк кода
5. Давать понятное описание: «Что должно получиться»

**Антипаттерны:**
- Миссия не вызывает ключевую конструкцию (например, миссия на `random.choice` без `random.choice()`)
- expected_output противоречит задаче

## 7. Attention checks (ловушки внимания)

Для уроков по базовым темам добавлять минимум один attention check:

| Конструкция | Attention check |
|---|---|
| print() | Забыл кавычки |
| Переменные | Пробел в имени, цифра в начале |
| Кавычки | Смешал одинарные и двойные |
| input() | input() возвращает строку |
| int(input()) | Забыл int() — сложение строк |
| if | `=` вместо `==`, пропущено `:` |
| if/else | Отступы, else на том же уровне |
| for | Забыл `:`, не тот диапазон range |
| while | Бесконечный цикл, забыл счётчик |
| Списки | Индекс с 0, а не с 1 |
| Функции | Забыл `()`, забыл `return` |
| Словари | Ключа нет — KeyError |
| try/except | Слишком широкий except |

## 8. Практика и проверка

1. **Quiz** — 1 вопрос на понимание главной идеи
2. **What Outputs** — предсказать поведение кода (1-2 варианта)
3. **Find Bug** — найти и исправить ошибку
4. **Mission** — написать код с нуля
5. **Practice Subtasks** (опционально) — 1-2 маленьких задания перед миссией

---

## 9. Запрещённые формулировки

- ❌ «Понял: ...» (всегда)
- ❌ «Осознал: ...» (всегда)
- ❌ «Ой-ой! Багус нашёл багус! Python в замешательстве!» (только 1 раз во всём курсе)
- ❌ «Хм! Багус одобряет эту ошибку!» (только 1 раз во всём курсе)
- ❌ «Ошибка — это тоже прогресс!» (только 1 раз во всём курсе)
- ❌ «Не переживай, с первого раза редко получается.» (сухая мотивация)
- ❌ «Идеального кода не бывает. Бывает код, который работает.» (клише)
- ❌ Сухие определения без аналогии
- ❌ Абстрактные объяснения «для взрослых» без бытового образа

## 10. Проверка перед выпуском

- [ ] Аналогия есть и раскрыта до кода
- [ ] Новичок звучит как ученик, а не как учитель
- [ ] Bagus говорит по делу, а не общие фразы
- [ ] «Почему это важно» есть
- [ ] Диалоги не повторяются между уроками
- [ ] Миссия проверяет тему урока
- [ ] Attention check есть для базовых конструкций
- [ ] Common mistakes релевантны теме
- [ ] Код в runnable_code выполним
- [ ] Понятно для 12-летнего новичка
