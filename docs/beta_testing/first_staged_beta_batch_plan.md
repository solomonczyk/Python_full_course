# First Staged Beta Batch Plan

## Goal
Run the first staged beta group with 5–10 real participants to validate the learning experience, test staged access mechanics, and gather actionable feedback before expanding the beta.

## Access Model
- **Default stage:** Part 1 (opened immediately with a valid beta code)
- **Open content:** Part 1 lessons (all 3 chapters, approximately 30 lessons)
- **Locked content:** Part 2 and Part 3 (accessible only after operator unlock)
- **Unlock condition:** Parent/operator submits feedback → operator reviews → operator executes unlock API

## Batch Size
- **Minimum:** 5 participants recruited
- **Target:** 7–8 participants
- **Maximum:** 10 participants

## Success Criteria

| Criterion | Target | Minimum |
|---|---|---|
| Participants recruited | 8 | 5 |
| Participants started Part 1 | 7 | 5 |
| Part 1 completed | 5 | 3 |
| Feedback received | 5 | 3 |
| Part 2 unlocked | 3 | 2 |
| No critical blockers | 0 | 0 |

## Metrics

| Metric | Measurement |
|---|---|
| **started Part 1** | Participant used beta code and began first lesson |
| **completed Part 1** | Participant reached the end of Part 1 content |
| **feedback submitted** | Feedback form received with meaningful content |
| **unlocked Part 2** | Operator executed unlock API for the participant |
| **asked for help** | Participant/parent contacted operator with questions |
| **abandoned** | Participant stopped without completing Part 1 and did not return |

## Participant Sourcing Plan

| Source | Target Count |
|---|---|
| Warm network (friends, acquaintances) | 3–4 |
| Teacher/tutor referrals | 1–2 |
| Parent community groups | 1–2 |
| Local Serbian/Ukrainian community | 1–2 |

## Timeline

| Week | Action |
|---|---|
| Week 1 | Outreach and screening |
| Week 1–2 | Issue beta codes, first sessions |
| Week 2–3 | Part 1 completion, feedback collection |
| Week 3 | Unlock decisions, Part 2 for early completers |
| Week 4 | Batch report, decision on next batch |

## Decision After Batch

Based on results, choose one:

- **Continue recruiting** — Batch completed successfully, expand to next 10–20 participants
- **Fix blockers** — Critical/major issues found, pause recruitment, fix product, restart
- **Update onboarding** — Participants struggled to start, improve first-time experience
- **Move to payment/legal** — Only after beta validates the product and all blockers are resolved

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Participants don't complete Part 1 | Medium | Low | Focus on engaged families, check in after 3 days |
| Technical blockers found | Medium | Medium | Have fix + redeploy plan ready (1–2 day turnaround) |
| Beta codes shared publicly | Low | High | Issue codes privately, monitor for abuse |
| Parents misunderstand beta scope | Medium | Medium | Clear screening, staged access explanation in invitation |
| Child cannot use independently | Low | Low | Screen for age/browser readiness, recommend parent supervision |
