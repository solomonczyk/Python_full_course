# Beta Recruitment Strategy

## Goal
Find the first 5–10 real staged beta participants for Python Quest.

## Target Participants
- **Children / beginners:** Age 9–14, approximate
- **Recommended age range:** 10–13 for independent use; 9–10 with parent supervision
- **Parent/operator role:** Parent registers interest, supervises setup, observes sessions, provides feedback
- **Required device/browser:** Modern laptop/desktop (Windows, macOS, Linux) with Chrome, Firefox, Edge, or Safari (latest versions). Tablets not supported yet.
- **Required time commitment:** 2–3 sessions of 20–40 minutes each to complete Part 1.

## Positioning
- **What we say:** "Python Quest is an interactive story-driven way to learn Python. We're running a small free beta — access opens one part at a time. Your feedback helps us make it better."
- **What we do not say:** "It's a fully finished product." "Your child will get a certificate." "This is a complete school curriculum."
- **Why staged beta access exists:** To release early, gather real feedback from real families, fix issues before opening more content, and avoid overwhelming young beginners with too much at once.

## Recruitment Channels

### Warm Network
- **Friends:** Personal messages to friends with children in the target age range
- **Parents you know:** Reach out to parents from school, extracurricular, or neighbourhood
- **Former colleagues:** Contact former colleagues who have school-age children
- **Local community:** Neighbourhood chats, parent meetups

### Parent Communities
- **Telegram:** Serbian programming parents chats, Ukrainian tech parent groups, Russian-speaking coding for kids channels
- **Facebook:** Parents Who Code, Kids Coding groups, Expat parenting in Serbia, Homeschool groups
- **Viber:** Local parent groups, school chat groups
- **Local Serbian / Ukrainian / Russian-speaking groups:** Community groups for families interested in tech education

### Teachers / Tutors
- **Informatics teachers:** Local school teachers who might recommend the course to interested students
- **Python tutors:** Individual tutors looking for structured materials for their students
- **Children IT clubs:** Small after-school coding clubs
- **Homeschool groups:** Families doing alternative education who want a structured programming module

## Recruitment Funnel
1. **Outreach message** → Contact parent/tutor via warm network or community group
2. **Parent interest** → Parent responds, asks questions
3. **Screening** → Operator runs through screening form (5–10 min call or text exchange)
4. **Invitation** → If fit confirmed, send invitation with beta explanation
5. **Beta code issued** → Generate and deliver unique beta code
6. **Child completes Part 1** → Child works through lessons at own pace
7. **Feedback collected** → Parent/operator fills feedback form
8. **Operator unlock decision** → Operator reviews feedback, decides on Part 2 unlock

## Success Criteria
- 5–10 participants recruited
- At least 5 start Part 1
- At least 3 complete Part 1
- Feedback received before unlock
- No personal child data collected
- No critical blockers found
- At least 2 participants ready for Part 2 unlock

## Stop Conditions
- **Parent expects finished product:** Stop. Explain beta scope again. If still expects finished product, do not proceed.
- **Parent refuses beta limitations:** Stop. Do not participate without staged access agreement.
- **Child cannot start without adult technical help:** Allow one assisted setup, but if child cannot proceed independently through lessons, pause participation.
- **Operator cannot manage codes safely:** If beta codes are leaked or operator cannot track distribution, pause recruitment.
- **Critical blocker identified in Part 1:** Pause recruitment until blocker is fixed.
