# Commercial FAQ

## What is Python Quest?
Python Quest is a story-driven, mission-based Python course for beginners aged 10–16. Instead of traditional video lessons, students learn by completing coding missions within a steampunk narrative, guided by characters Ва (mentor), Новичок (learner), and Багус (bug creature). Every lesson ends with writing real Python code.

## Who is it for?
Children and absolute beginners aged 10–16 who want to learn programming from zero. It is also suitable for older teenagers and adults who prefer a more engaging, story-driven introduction to Python over dry textbooks.

## Is it for complete beginners?
Yes. Python Quest assumes zero programming experience. The first lessons explain what a variable is, what data types are, and how to write your first line of code. No prior knowledge is required.

## Does the child need to install Python?
No. Everything runs in the browser. The student writes code in an embedded editor, and the Mission Checker evaluates it instantly. No installations, no PATH configuration, no environment setup.

## Is this a full Python course?
Python Quest covers Python fundamentals: variables, data types, conditions, loops (for/while), lists, functions with parameters/return values, dictionaries, and basic algorithmic thinking. It is a complete introductory course but does not cover advanced topics (classes, modules, file I/O, libraries, web frameworks).

## What does beta mean?
Beta means the product is feature-complete and production-ready but has not yet been tested with real students at scale. During beta, access is free, and we actively collect feedback from students, parents, and operators to identify remaining issues and improvement opportunities.

## What will be improved after beta?
Based on beta feedback, we may adjust:
- Lesson wording and dialogue length.
- Mission difficulty curve and hint quality.
- Onboarding first-time experience.
- UI animations and progress visualization.
- Any content that proves confusing or demotivating.

## Is there homework?
No. Python Quest is designed for self-paced, session-based learning. There are no mandatory homework assignments, though students are welcome to experiment with the concepts they learn on their own.

## How is progress tracked?
Progress is shown visually: completed lessons display a completion marker on the quest grid, quests show overall completion, and the skill tree reveals which Python abilities have been unlocked. During beta, progress is session-based (not permanently stored across browser restarts unless an account system is added later).

## What if the student makes mistakes?
Mistakes are a core part of the learning design. When code is incorrect, the Mission Checker provides a friendly error message, and characters offer hints. No score is deducted, no penalty is applied. The student tries again until they succeed.

## Is payment available now?
No. The beta is completely free. Payment will be introduced only after a separate payment and legal implementation layer is completed, which is outside the scope of the current phase.

## What is not ready yet?
- Mass paid launch (requires payment layer, legal, and privacy review).
- Student/parent dashboard with persistent accounts.
- School/institution management tools.
- Certificate generation.
- Automated email notifications and progress reports.
- Advanced topics beyond Python fundamentals.
