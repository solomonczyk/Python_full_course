# Python Quest Landing Structure

## Hero Section
- **Headline:** Learn Python. Complete Missions. Save the World.
- **Subheadline:** A story-driven coding adventure where children aged 10–16 learn real Python through interactive missions, character guidance, and instant feedback.
- **Primary CTA:** "Start Quest" (opens free/demo access)
- **Secondary CTA:** "See How It Works" (scrolls to How It Works section or opens short explainer)

## Problem Section
**Headline:** Most children's Python courses feel like school.  
**Body:**
- Passive video lessons don't teach coding — they teach watching.
- Abstract examples without a story lose attention fast.
- Mistakes feel like failure instead of part of learning.
- Parents want their child to learn real skills, not just click through gamified fluff.

## Solution Section
**Headline:** Python Quest is different.  
**Body:**
- Every concept is part of a steampunk adventure with characters who guide, question, and celebrate.
- Students write real Python code from the very first mission.
- Mistakes are met with friendly hints, not red crosses.
- Progress is visible: quest maps, skill unlocks, completion markers.
- By the end, your child can write real Python scripts — not just drag blocks.

## How It Works
1. **Start quest** — One click, no setup, no install. The story begins.
2. **Learn concept through analogy** — Ва explains Python ideas using everyday things: variables as labelled chests, conditions as forks in the road.
3. **Complete mission** — Write real code. The Mission Checker gives instant friendly feedback.
4. **Get feedback** — Pass or stuck? hints and encouragement from characters.
5. **Unlock next ability** — Complete lessons to unlock new Python skills and continue the story.
