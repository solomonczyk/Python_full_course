# Python Quest Beta Package

## Product Summary
- **Product name:** Python Quest
- **Current status:** Production accepted — commercial beta ready
- **Beta goal:** Validate that real beginner students (ages 10–16) can understand Python fundamentals through a story-driven, mission-based game journey without external help, and collect feedback to polish the product toward mass paid launch.

## Who It Is For
- **Target learner:** Children and absolute beginners who want to learn programming but find traditional courses boring or intimidating.
- **Age range:** 10–16 years old (younger learners may need parent assistance).
- **Starting level:** Absolute zero — no prior programming experience required.
- **Parent/operator role:** Parent or tutor sets up the environment (opens the landing page), observes progress, provides encouragement, and submits feedback. No technical setup required beyond a modern browser.

## What Is Included
- **Available lessons:** 92 interactive lessons organised across 6 quests, plus 9 recap lessons.
- **Game mechanics:** Steampunk-themed narrative, character dialogues (Ва, Новичок, Багус), visual mission cards, progress tracking with completion dots.
- **Missions:** Hands-on coding missions embedded in every lesson — students write real Python code and receive instant feedback from the Mission Checker.
- **Progress tracking:** Visual lesson grid, quest completion indicators, skill tree unlock markers.
- **Characters:**
  - **Ва** — wise mentor who explains concepts through analogies and guides the journey.
  - **Новичок** — the protagonist learner, asks questions students would ask, normalises mistakes.
  - **Багус** — a chaotic but helpful bug-like creature, demonstrates what happens when code goes wrong.

## Expected Result For Student
- **What student should understand:** Core Python concepts — variables, data types, conditions, loops, functions, lists, dictionaries, and basic algorithmic thinking.
- **What student should be able to do:** Write small working Python scripts, understand error messages, fix mistakes independently, explain their code to a peer or parent, and feel confident to continue learning.

## Beta Limitations
- **What is not final:** Some mission text length, progression speed across Part 3 quests, and character dialogue density may be adjusted after feedback.
- **What is not included yet:** Mass paid access, student/parent dashboards, automated email journeys, certificate generation, school/institution management.
- **What may change after feedback:** Lesson pacing, hint wording, mission difficulty curve, onboarding first-time experience, and character dialogue volume.

## Feedback We Need
- **Confusing lessons:** Which lesson or mission made the student stop or ask for help.
- **Hard missions:** Which mission blocked progress despite hints.
- **UI blockers:** Where the student clicked and nothing happened, or the interface was unclear.
- **Motivation drop points:** Where the student lost interest and wanted to stop.
- **Parent feedback:** Does the parent understand what the product does and what progress means? Is the product valuable enough to pay for?
