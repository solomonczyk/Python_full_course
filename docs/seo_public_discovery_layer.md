# SEO & Public Discovery Layer — Python Quest

## Overview

Python Quest is a client-side SPA (Vite + React + React Router v6) hosted on
Vercel.  Because there is no server-side rendering (SSR), traditional search
engine crawlers that do not execute JavaScript cannot discover route content.

This document describes the SEO layer added to make the site publicly
discoverable despite the SPA limitation.

---

## Architecture

```
SPA (Vite / React Router)
├── index.html          ← static meta tags (title, OG, Twitter, canonical)
├── robots.txt          ← crawler directives
├── sitemap.xml         ← static sitemap (generated by script)
├── src/App.tsx         ← routes + MetaRouter (dynamic meta on route change)
├── src/hooks/usePageMeta.ts  ← client-side meta tag updater
├── src/pages/CourseCatalogPage.tsx  ← public /course
└── src/pages/LessonPreviewPage.tsx  ← public /lesson/:id/preview
```

## Public Routes

| Route | Description | Onboarding Required |
|-------|-------------|-------------------|
| `/` | Homepage (curriculum) | ✅ Yes (learning flow) |
| `/course` | Public lesson catalog | ❌ No |
| `/lesson/:id/preview` | Public lesson preview | ❌ No |
| `/lesson/:id` | Full lesson (learning flow) | ✅ Yes |
| `/onboarding` | Onboarding flow | ❌ No (it IS onboarding) |

## SEO Metadata

### Static (index.html)
- `title` — "Python Quest — игровой курс Python для новичков"
- `description` — Full course description (Russian)
- OpenGraph tags — `og:title`, `og:description`, `og:type`, `og:image`, `og:locale`
- Twitter card tags — `twitter:card`, `twitter:title`, `twitter:description`
- Canonical URL — `https://python-quest.app/`
- Language — `ru`
- Keywords — Python, обучение, курс, программирование, новички

### Dynamic (client-side via `usePageMeta` hook)
- **Homepage / lessons / reviews**: Uses `PUBLIC_ROUTES['/']` metadata
- **Course catalog** (`/course`): "Каталог уроков Python Quest" title
- **Lesson preview** (`/lesson/:id/preview`): Dynamic title from lesson data
- **Onboarding** (`/onboarding`): "Вступление" title

### `usePageMeta` hook
```typescript
usePageMeta({
  title: 'Каталог уроков',
  description: '...',
  ogTitle: '...',
  ogDescription: '...',
  canonical: 'https://python-quest.app/course',
})
```

Updates: `document.title`, `meta[name="description"]`, OG tags,
Twitter card tags, canonical `<link>`.  Calls `applyPageMeta()` on mount;
restoration happens on the next route change.

## Onboarding Gate

The onboarding gate (`pq_onboarding_done` in localStorage) **no longer blocks**
public routes:

- `/course` and `/lesson/:id/preview` are defined **outside** the onboarding
  check in the React Router `<Routes>` tree
- Only the catch-all `/*` route (which covers `/`, `/lesson/:id`,
  `/review/:id`) checks for the onboarding key
- The `/completion` page and the existing learning flow remain unchanged

## Static Files

### `public/robots.txt`
- Allows all crawlers (`User-agent: *`)
- Disallows `/onboarding` and `/completion`
- References sitemap at `https://python-quest.app/sitemap.xml`

### `public/sitemap.xml`
- Generated by `scripts/generate_sitemap.py`
- Contains 94 URLs: 2 static (`/`, `/course`) + 92 lesson previews
- Uses correct `https://python-quest.app` base URL
- Set `changefreq` and `priority` per route type

### Sitemap generation
```bash
python scripts/generate_sitemap.py
```
The script reads lesson data from `backend/app/data/lessons.json` and outputs
`sitemap.xml` to `frontend/public/`.  It should be re-run when lessons are
added, removed, or renamed.  A CI step or Vercel build hook can automate this.

## SPA Limitation (Documented)

**This project does NOT have SSR/SSG.**  Vite + React build to static files
(the traditional SPA model).  The meta tags in `index.html` are the only
content search engines see on first HTTP request without JS execution.

To fully solve this, the app would need:
- Next.js or similar SSR framework
- Or Vercel's ISR / Edge Functions for per-route HTML
- Or a pre-rendering service (Prerender.io, Rendertron)

The current layer achieves the **maximum possible SPA-compatible SEO**:
static meta + sitemap + robots.txt + public routes + client-side meta
updates.  This is sufficient for:
- Google (which executes JS for indexing)
- Social media link previews (OG/Twitter tags in `index.html`)
- Users navigating directly to `/course` or a preview page

## Accessibility

Improvements made:
- Navigation landmarks (`<nav>` with `aria-label` on TopNav)
- Sidebar overlay: `role="presentation"` + `aria-hidden="true"`
- Public pages: single `<h1>`, proper `<h2>/<h3>` hierarchy
- Breadcrumb `<nav aria-label="Breadcrumb">` on preview pages
- Loading/error/empty states with clear text
- All `<img>` tags have meaningful `alt` text
- Buttons vs links used semantically (`<Link>` for navigation, `<button>` for actions)

## Carryover Fix

**Bug**: `mission.py` line ~121 appended all hints starting with `🔧`, `❗`,
or `⛔` to the `errors` list.  This caused constructive hints (e.g. "Нужно
создать переменную") to be returned as errors, which the frontend rendered
as severe Bagus error messages.

**Fix**: Only `⛔` (safety violations) and `❗` (syntax errors) are added to
errors.  `🔧` (constructive hints) and `⚠️` (warnings) remain in the `hints`
list only.
